# 新建 STM32 项目须知

## 1. 晶振频率（时钟配置）

STM32 的系统时钟（SYSCLK）通过 PLL 倍频得到，PLL 的输入是外部晶振（HSE）。
**PLL_M 分频系数必须等于晶振频率（MHz），否则 PLL 输出不对。**

例：8MHz 晶振 → PLL_M = 8 → SYSCLK = 168MHz
例：25MHz 晶振 → PLL_M = 25 → SYSCLK = 168MHz

如果配错，芯片可能跑在默认 16MHz（HSI），导致定时器、串口波特率、SysTick 延时全部不准。

| 已下载芯片包 | 时钟来源 | 说明 |
|---|---|---|
| 已下载且已选择 | 芯片包中的 `system_stm32xxx.c` | 需确认 PLL_M 匹配晶振 |
| 未下载 | 应用内置的 `system_stm32f4xx.c`（仅 F4） | 已预设 8MHz 晶振 |
| 其他系列（F7/H7 等）无内置 | 弱定义 SystemInit（空函数） | 跑默认 16MHz，需自行配置 |

**推荐做法：** 新建项目后先检查 `DRIVE/CMSIS/system_stm32f4xx.c` 中 `PLL_M` 是否等于你的晶振 MHz 数。

## 2. 外设驱动库选择

新建项目时可选择以下四种：

| 选项 | API 风格 | 来源 | 适用于 |
|---|---|---|---|
| 标准外设库（StdPeriph） | `GPIO_Init()`、`RCC_AHB1PeriphClockCmd()` 等 | 应用内置（仅 F4） | 旧项目迁移、正点原子例程 |
| HAL 库 | `HAL_GPIO_Init()`、`__HAL_RCC_GPIOx_CLK_ENABLE()` 等 | 需已下载芯片包 | STM32CubeMX 生成的项目 |
| LL 库 | `LL_GPIO_Init()`、`LL_RCC_EnableAHB1Periph()` | 需已下载芯片包 | 轻量级 HAL 替代 |
| 无（仅 CMSIS） | 寄存器操作 | — | 最小项目 |

StdPeriph 和 HAL/LL 的 API **完全不兼容**，选错无法编译。

## 3. 项目目录结构

创建成功后项目结构如下：

```
项目名称/
├── USER/                ← 用户代码（main.c 写在这里）
├── DRIVE/
│   ├── CMSIS/           ← CMSIS 头文件 + system_stm32xxx.c
│   ├── FWLib/
│   │   ├── inc/         ← 外设库头文件（选 StdPeriph/HAL/LL 时有）
│   │   └── src/         ← 外设库源码
│   └── stm32_mcu        ← 芯片型号（自动写入，供构建脚本读取）
└── codex_build/         ← 构建产物（脚本自动生成）
```

---

## 4. LED / 外设引脚

模板 main.c 默认使用 PC13（常见板载 LED）。如果实际 LED 不在 PC13，需自行修改：

```c
#define LED_PIN   GPIO_Pin_13
#define LED_PORT  GPIOC
```
